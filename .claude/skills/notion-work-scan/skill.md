# Notion Work Scan — 노션 업무 항목 구조화 수집 스킬

노션 워크스페이스에서 업무 관련 데이터베이스와 페이지를 탐색하여 작업 현황을 담당자별/프로젝트별/팀별로 구조화 수집하는 절차적 가이드.

## 수집 워크플로우

### Step 1: 워크스페이스 탐색

검색 키워드 목록을 사용하여 노션을 탐색한다. 키워드가 별도로 주어지지 않았으면 아래 기본 키워드를 순서대로 사용한다:

```
기본 검색어: "업무", "작업", "프로젝트", "할일", "스프린트", "마일스톤", "태스크", "task", "TODO", "진행"
```

각 검색어로 `notion-search` 호출:
- `query_type: "internal"` 사용
- `page_size: 25` (최대)로 설정하여 누락 최소화
- 검색 결과에서 데이터베이스와 페이지를 구분하여 기록

### Step 2: 데이터베이스 스키마 분석

발견된 데이터베이스 각각에 대해:

1. `notion-fetch`로 데이터베이스 스키마 확인
   - 속성 이름과 타입 파악 (상태, 담당자, 날짜, 선택 등)
   - data_source_id 확보

2. 핵심 속성 매핑 — 다양한 이름 패턴을 인식한다:

   | 필드 | 한국어 패턴 | 영어 패턴 |
   |------|-----------|----------|
   | 상태 | 상태, 진행상황, 단계 | Status, Stage, Phase |
   | 담당자 | 담당자, 담당, 작업자, 배정 | Assignee, Owner, Assigned |
   | 마감일 | 마감일, 기한, 날짜, 마감 | Due Date, Deadline, Due |
   | 프로젝트 | 프로젝트, 소속 프로젝트 | Project |
   | 팀 | 팀, 부서, 그룹 | Team, Department, Group |
   | 우선순위 | 우선순위, 중요도 | Priority, Importance |

3. 팀/프로젝트 속성이 없는 경우 대체 추론:
   - 데이터베이스 이름에서 프로젝트명 추출 (예: "ProjectX 작업 보드" → 프로젝트: ProjectX)
   - 상위 페이지 제목에서 팀/프로젝트 유추
   - 추론한 경우 비고에 "(추정)" 표기

### Step 3: 작업 항목 수집

활성 작업 위주로 수집:
- "완료", "Done", "Closed" 상태는 제외 (최근 1주일 내 완료 건은 포함)
- "진행 중", "In Progress", "할 일", "To Do", "대기" 등 활성 상태 우선

각 작업에서 추출할 필드:
1. **작업명** (필수)
2. **상태** (필수)
3. **담당자** (필수 — 없으면 "미배정")
4. **팀** (가능하면 — 없으면 추론 시도)
5. **프로젝트** (가능하면 — 없으면 추론 시도)
6. **마감일** (있으면)
7. **우선순위** (있으면)
8. **비고** (특이사항)

### Step 4: 페이지 기반 업무 수집

데이터베이스가 아닌 일반 페이지 중 업무 관련 내용:

1. `notion-fetch`로 페이지 내용 확인
2. 체크리스트, TODO 항목, 회의록 내 액션 아이템 추출
3. 하위 페이지가 있으면 제목 수준까지 확인
4. 추출된 항목도 가능한 한 동일 필드 구조로 정리

### Step 5: 긴급/임박 항목 분류

수집된 전체 항목에서 (오늘 날짜 기준):
- **긴급**: 마감일이 오늘~3일 이내
- **주의**: 마감일이 4~7일 이내
- **지연**: 마감일이 이미 지남
- **블로커**: 상태가 "블로킹", "대기", "Blocked" 등

### Step 6: 구조화 정리

수집된 모든 데이터를 에이전트 정의의 출력 형식에 맞춰 정리한다:
1. **전체 작업 목록** 테이블 (모든 작업을 하나의 표로)
2. **팀별 분류** (팀 이름을 섹션 헤더로)
3. **프로젝트별 분류** (프로젝트 이름을 섹션 헤더로)
4. **긴급/임박 항목** (위의 분류 기준)
5. **발견된 키워드 목록** (프로젝트명, 팀원명, 팀명 — slack-scanner가 활용)

출력 파일: `_workspace/01_notion_collector_work_items.md`

## 검색 최적화 팁

- 첫 검색에서 결과가 적으면 영어/한국어 키워드를 모두 시도
- 팀명이나 프로젝트명을 알면 해당 키워드로 추가 검색
- 데이터베이스 내 검색은 `data_source_url` 파라미터를 활용하여 범위를 좁힘
- 같은 작업이 여러 검색에서 중복 발견되면 한 번만 기록 (중복 제거)
