# Slack Discussion Scan — 슬랙 업무 대화 구조화 수집 스킬

노션의 AI 검색(연결된 소스)을 활용하여 슬랙에서 업무 관련 대화를 탐색하고, 프로젝트/팀/담당자별로 연결하여 정리하는 절차적 가이드.

## 수집 워크플로우

### Step 1: 검색 키워드 준비

오케스트레이터로부터 전달받은 키워드 목록을 사용한다. 추가로 아래 기본 키워드도 함께 사용:

```
기본 검색어: "진행 상황", "업데이트", "리뷰", "피드백", "확인 부탁", "블로커", "일정", "배포", "회의"
```

notion-collector가 수집한 프로젝트/작업명/팀원명이 있으면 해당 이름으로 추가 검색한다.

### Step 2: 슬랙 대화 검색

각 검색어로 `notion-search` 호출:
- `content_search_mode: "ai_search"` 설정 (연결된 소스 포함)
- `page_size: 15`로 설정
- 검색 결과 중 슬랙 출처(URL에 slack이 포함되거나 소스 타입이 Slack)인 것만 필터링

슬랙 결과가 아닌 노션 결과만 나오는 경우:
- 슬랙 연동이 되어 있지 않을 수 있음 → "슬랙 연결 소스 없음" 보고

### Step 3: 대화 내용 분석 및 분류

발견된 각 슬랙 대화에서 추출할 정보:

**기본 정보**: 채널명, 날짜, 참여자

**패턴별 분류**:

| 분류 | 인식 패턴 | 추출 내용 |
|------|----------|----------|
| 의사결정 | "~로 하자", "~로 결정", "approved", "확정", "합의" | 결정 내용, 참여자, 날짜 |
| 액션 아이템 | "~해주세요", "~부탁", "~까지", "@멘션 + 동사" | 요청자, 담당자, 내용, 기한 |
| 블로커/이슈 | "막혀있", "대기 중", "이슈", "문제", "blocked", "장애" | 내용, 심각도, 관련 프로젝트 |
| 피드백 요청 | "~인가요?", "확인 부탁", "리뷰 요청", "의견 주세요" | 요청자, 내용, 대상 |

**프로젝트/팀 연결**:
- 채널명에서 프로젝트/팀 유추 (예: #dev-projectX → 프로젝트: ProjectX, 팀: 개발팀)
- 대화 내용에서 프로젝트명/팀명 키워드 감지
- 확실하지 않으면 "(추정)" 표기

### Step 4: 노션 미등록 작업 감지

슬랙에서 업무로 논의되고 있으나 notion-collector의 작업명과 매칭되지 않는 항목을 식별한다:

감지 기준:
- 구체적인 작업/기능/버그에 대한 논의인데 노션 키워드 목록에 없는 경우
- "이거 노션에 등록했어?", "티켓 만들어야 할 것 같아" 같은 직접적 언급
- 명확한 액션 아이템이 있지만 노션 작업과 매칭 안 되는 경우

### Step 5: 우선순위 분류

수집된 대화를 중요도 순으로 정렬:
1. **긴급**: 블로커, 긴급 요청, 장애 관련
2. **중요**: 의사결정 사항, 마감 임박 관련
3. **참고**: 일반 업데이트, 정보 공유

### Step 6: 구조화 정리

수집된 모든 데이터를 에이전트 정의의 출력 형식에 맞춰 정리한다:
1. **의사결정 사항** 테이블
2. **액션 아이템** 테이블
3. **블로커/이슈** 테이블
4. **프로젝트별 논의 요약**
5. **노션 미등록 추정 작업**
6. **대기 중인 피드백/요청**

출력 파일: `_workspace/01_slack_scanner_discussions.md`

## 검색 팁

- 검색어는 짧고 구체적일수록 좋음 (예: "배포 일정" > "다음 주 배포 관련 논의")
- 사람 이름으로 검색하면 해당 인물 관련 대화를 빠르게 찾을 수 있음
- 최근 대화가 가장 가치있으므로 날짜 필터 활용 권장
- 같은 스레드가 여러 검색에서 중복 발견되면 한 번만 기록 (중복 제거)
